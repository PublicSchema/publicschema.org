# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in PublicSchema, please report it responsibly.

**Email:** security@publicschema.org

Please include:

- A description of the vulnerability
- Steps to reproduce the issue
- The potential impact

We will acknowledge receipt within 48 hours and aim to provide a resolution timeline within one week.

## Scope

PublicSchema is a static vocabulary website. The primary security concerns are:

- Integrity of vocabulary definitions and JSON-LD contexts (supply chain)
- Correctness of Verifiable Credential schemas
- Website integrity (XSS, content injection)

## Supported Versions

Only the latest version published at publicschema.org is supported.
